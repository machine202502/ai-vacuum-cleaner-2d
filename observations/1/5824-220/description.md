## Запуск обучения

```bash
python training/train_torch.py random --episodes 10000 --episode-time-sec 220 --gru --workers 8 --plot-every 8 --save-every 100 --curriculum
```

- Комнаты: `random` → циклически `rooms/random*.json`, смена комнаты каждые 20 эпизодов.
- Алгоритм: PPO (actor–critic), GAE, TBPTT для GRU; 8 параллельных воркеров = 8 эпизодов на батч-апдейт.
- Curriculum (дефолты CLI): длина эпизода с 20 с +10 с каждые 200 эпизодов, потолок 220 с; шаг среды 60 FPS → до 13200 шагов/эпизод на максимуме.
- Остальное по умолчанию: `lr=1e-3`, `gamma=0.99`, `hidden=128`, `encoder_layers=2`, Adam + cosine LR к `lr*0.01`.

## Архитектура

- Вход: **8** чисел (3 ИК до стены, sin/cos угла, одометр суммарный и за шаг, норм. время в эпизоде) — см. `VacuumEnv.OBS_DIM`.
- Сеть: **Encoder** — 2×(Linear+ReLU) `8 → H → H` (H=128).
- **GRU(H→H)** один слой, далее две головы:
  - **Actor:** Linear→ReLU→Linear → **8** логитов;
  - **Critic:** Linear→ReLU→Linear → **1** value.
- Эпизод: скрытое состояние GRU сбрасывается в 0 на каждом `reset`.

## Награды и штрафы (каждый шаг среды)

- Базово: **−0.1** за шаг (`STEP_PENALTY`).
- Новая клетка при движении **вперёд** (щётка): **+1.0** за 1-е посещение; **−0.1 / −0.2 / −0.3** за 2–4-е; с 5-го ещё **−0.1** за каждое лишнее.
- Доп. к награде за клетку при **строго вперёд** без поворота: **+0.05** за клетку; если рядом уже была убранная клетка — ещё **+0.05** (раз на клетку).
- **Назад:** посещения клеток по карте не начисляются (только штрафы за шаг/простой/удар).
- Нет движения (`encoder_delta == 0`): **−0.2** (`IDLE`); если при этом был вперёд/назад в стену — ещё **−1.0** (`COLLISION`).
- Долго без движения: каждые **120** таких шагов **−2.0**; **300** подряд — конец эпизода.
