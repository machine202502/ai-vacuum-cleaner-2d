## Запуск обучения

```bash
python training/train_torch.py random --episodes 5000 --episode-time-sec 440 --gru --workers 4 --plot-every 8 --save-every 100 --curriculum --hidden 234
```

- Комнаты: `random` → циклически `rooms/random*.json`, смена комнаты каждые 20 эпизодов.
- Алгоритм: PPO (actor–critic), GAE, TBPTT для GRU; 4 параллельных воркера = 4 эпизода на батч-апдейт.
- Curriculum (дефолты CLI): длина эпизода с 20 с +10 с каждые 200 эпизодов, потолок 440 с; шаг среды 60 FPS → до 13200 шагов/эпизод на максимуме.
- Остальное по умолчанию: `lr=1e-3`, `gamma=0.99`, `encoder_layers=2`, Adam + cosine LR к `lr*0.01`. Явно задано: **`hidden=234`**.

## Архитектура

- Вход: **6** чисел (3 ИК до стены, sin/cos угла, одометр **за шаг** — нормированный `encoder_delta`) — см. `VacuumEnv.OBS_DIM`.
- **Решение по наблюдению:** из вектора убраны два канала — накопленный одометр за эпизод и псевдо-«время» (нормировка по 24 ч давала почти константу и дублировала счёт шагов). Они давали **шум и насыщение** (суммарный путь упирался в 1.0), пользы для политики мало, особенно при GRU. Остаётся только **delta за шаг** как явный сигнал «ехали / встали у стены».
- Сеть: **Encoder** — 2×(Linear+ReLU) `6 → H → H` (H=234).
- **GRU(H→H)** один слой, далее две головы:
  - **Actor:** Linear→ReLU→Linear → **8** логитов;
  - **Critic:** Linear→ReLU→Linear → **1** value.
- Эпизод: скрытое состояние GRU сбрасывается в 0 на каждом `reset`.

## Награды и штрафы (каждый шаг среды)

- Базово: **−0.1** за шаг (`STEP_PENALTY`).
- Новая клетка при движении **вперёд** (щётка): **+1.0** за 1-е посещение; **−0.1 / −0.2 / −0.3** за 2–4-е; с 5-го ещё **−0.1** за каждое лишнее.
- Доп. к награде за клетку при **строго вперёд** без поворота: **+0.05** за клетку; если рядом уже была убранная клетка — ещё **+0.05** (раз на клетку).
- **Назад:** посещения клеток по карте не начисляются (только штрафы за шаг/простой/удар).
- Нет движения (`encoder_delta == 0`): **−0.2** (`IDLE`); если при этом был вперёд/назад в стену — ещё **−1.0** (`COLLISION`).
- Долго без движения: каждые **120** таких шагов **−2.0**; **300** подряд — конец эпизода.

## Маленькие комнаты (для следующих итераций)

В **узкой** комнате агент быстро закрывает все клетки, дальше при длинном лимите времени остаётся езда по кругу: **повторные посещения и −0.1/шаг** штрафуют, но эпизод всё равно даёт **длинный хвост отрицательного return** → лишняя **дисперсия** градиентов PPO, смешение с «нормальными» эпизодами. Частично гасится сменой `random*` каждые **20** эп. и батчем из **4** воркеров. Идеи на потом: ранний `done` при полном покрытии, отбор/стратификация комнат по числу достижимых ячеек, иначе балансировать повторы.
